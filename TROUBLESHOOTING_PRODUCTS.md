# 🔧 استكشاف الأخطاء - نظام إدارة المنتجات

## المشكلة: البيانات الجديدة لا تظهر في صفحة العميل

إذا أضفت بيانات جديدة في لوحة الأدمن (مثل الكمية والسعر والإعدادات) ولم تظهر في صفحة المتجر للعميل، اتبع هذه الخطوات:

---

## ✅ الحل السريع

### الخطوة 1: مسح البيانات المحفوظة

1. اذهب إلى **Admin Panel → Product Management**
2. اضغط على زر **Reset** (مفتاح الحديد مع ثلاث خطوط)
3. اضغط **تأكيد**
4. الصفحة ستُعاد تحميل تلقائياً

### الخطوة 2: مسح متصفح Cache (اختياري)

1. افتح **Developer Tools** (F12)
2. اذهب إلى **Application → Local Storage**
3. احذف `products-storage` و `products-storage` المرتبط بـ URL الخاص بك
4. أغلق المتصفح وافتحه مرة أخرى

### الخطوة 3: أضف/عدّل منتج جديد

1. في **Product Management** → **Add Product** أو **Edit**
2. ملء جميع الحقول الجديدة:
   - **إعدادات المنتج**: حالة التوفر، الرؤية
   - **الكمية والتسعير**: الحد الأدنى والأقصى
   - **جدولة الظهور**: إذا كنت تريد جدولة
   - **إدارة المخزون**: إذا كنت تريد تتبع
3. اضغط **Preview** للتحقق من الحالة النهائية
4. اضغط **Save Product**

---

## 📊 البيانات المسموح بإضافتها

| الفئة              | الحقول                                                                                              |
| ------------------ | --------------------------------------------------------------------------------------------------- |
| **إعدادات المنتج** | `productStatus`, `isVisibleInStore`, `showWhenUnavailable`, `pauseSales`, `pauseReason`             |
| **الجدولة**        | `enableSchedule`, `scheduledStartAt`, `scheduledEndAt`, `scheduleVisibilityMode`                    |
| **الكميات**        | `minimumOrderQty`, `maximumOrderQty`, `stepQty`                                                     |
| **المخزون**        | `trackInventory`, `stockQuantity`, `lowStockThreshold`, `hideWhenOutOfStock`, `showOutOfStockLabel` |

## 🔍 كيفية التحقق من البيانات

### في Admin Panel:

1. عند إضافة/تعديل منتج، تظهر قسمة **Preview** في الأسفل
2. توضح **كيف سيبدو المنتج للعميل**
3. تشمل:
   - هل ظاهر أم لا
   - هل قابل للشراء أم لا
   - ما هي الشارة المعروضة
   - ما هو النص المساعد

### في متصفح العميل:

1. افتح **Developer Console** (F12)
2. في **Console** ستظهر رسالة:
   ```
   ✅ Products loaded: { name: "...", minimumOrderQty: 1, ... }
   ```
3. تحقق من أن البيانات الجديدة موجودة

## 🐛 استكشاف الأخطاء الشائعة

### ❌ المشكلة: لا تظهر أي بيانات جديدة

**الحل:**

- قم بـ **Reset** من Admin Panel
- امسح localStorage من Developer Tools
- أعد تحميل الصفحة (Ctrl+Shift+R)

### ❌ المشكلة: السعر لا يظهر

**الحل:**

- تأكد من ملء حقل **Base Price** في الكمية والتسعير
- إذا كان `syncPriceWithProvider = true`، اضغط **مزامنة الآن**
- تأكد من أن `enableManualPrice = false` إذا كنت تريد السعر المزامن

### ❌ المشكلة: الحد الأدنى للطلب لا يظهر

**الحل:**

- تأكد من ملء **minimumOrderQty**
- القيمة يجب أن تكون > 1 لتظهر في الشارة
- تأكد من عدم قيمة معطلة في المخزون

### ❌ المشكلة: منتج موقوف مؤقتاً لا يظهر

**الحل:**

- تأكد من تفعيل **إيقاف البيع مؤقتاً**
- تأكد من تفعيل **إظهار المنتج في المتجر**
- إذا كان الحالة = "غير متوفر"، فعّل **إظهاره كمعطل**

## 💾 حول حفظ البيانات

البيانات تُحفظ في 3 طرق:

1. **LocalStorage** - للتطوير المحلي (في المتصفح)
2. **Zustand Store** - الذاكرة الفورية
3. **API Server** (اختياري) - عند توصيل Backend حقيقي

### عند إضافة منتج:

```
Admin Form → handleProductSubmit → useMediaStore.addProduct → API + LocalStorage
```

### عند تحميل صفحة جديدة:

```
LocalStorage → Zustand Store → React Components → العرض على الشاشة
```

## 🚀 خطوات التصحيح النهائية

إذا استمرت المشكلة:

1. **مسح كل شيء:**
   - افتح Console (F12)
   - اكتب: `localStorage.clear()`
   - اضغط Enter
   - أعد تحميل الصفحة

2. **مراجعة الكود:**
   - تحقق من أن حقول productForm في AdminProducts تحتوي على الحقول الجديدة
   - تأكد من أن payload في handleProductSubmit يشمل جميع الحقول

3. **اختبر منتج جديد:**
   - أضف منتج جديد بكامل البيانات
   - تحقق من Preview
   - احفظ وتحقق من الظهور في المتجر
